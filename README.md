AI Task Planner - Интеллектуальный планировщик задач
 Обзор проекта
AI Task Planner — это веб-приложение для управления задачами с интегрированным AI-ассистентом на основе Yandex GPT. Проект разрабатывается поэтапно от простого MVP до полноценного интеллектуального помощника.

 Текущий статус: Этап 4/6
 MVP планировщик задач (CRUD операции)
 Красивый интерфейс с анимациями и уведомлениями
 Полная интеграция фронтенда и бэкенда
 Готовность к AI-интеграции (заглушки на месте)
 В разработке: AI-ассистент с Yandex GPT API

🛠 Технологический стек
Backend (FastAPI)
Python 3.9+

FastAPI (веб-фреймворк)

SQLAlchemy (ORM для работы с БД)

Frontend (Vue.js 3)
Vue.js 3 (Composition API)

Vue Router

Axios (HTTP клиент)

CSS3 с анимациями и flexbox/grid

Vite (сборщик проекта)

AI интеграция (готовится)
Yandex GPT API

Yandex Cloud

 Быстрый старт
Предварительные требования
Python 3.9 или выше

Node.js 16 или выше

npm или yarn

1️ Клонирование репозитория
bash
git clone <url-репозитория>
cd ai-task-planner
2️ Запуск бэкенда
Шаг 1: Настройка виртуального окружения
bash
# Перейдите в папку бэкенда
cd backend

# Создайте виртуальное окружение
python3 -m venv venv

# Активируйте виртуальное окружение:
# Для Windows:
venv\Scripts\activate
# Для Linux/Mac:
source venv/bin/activate
Шаг 2: Установка зависимостей Python
bash
pip install -r requirements.txt
Шаг 3: Запуск сервера
bash
python run.py
 Бэкенд запущен! Доступен по адресу: http://localhost:8000

Проверка работы бэкенда:
Документация API: http://localhost:8000/docs

Проверка здоровья: http://localhost:8000/

Список задач: http://localhost:8000/tasks

3️ Запуск фронтенда
Шаг 1: Установка зависимостей Node.js
bash
# Откройте новый терминал
cd frontend

# Установите зависимости
npm install
Шаг 2: Запуск сервера разработки
bash
npm run dev
 Фронтенд запущен! Доступен по адресу: http://localhost:5173

 Использование приложения
Основные функции
1. Создание задачи
Введите заголовок задачи в форме вверху страницы

Добавьте описание (опционально)

Нажмите "Создать задачу"

✅ Уведомление об успешном создании

2. Редактирование задачи
Нажмите ✏️ на карточке задачи

Измените заголовок, описание или статус

Нажмите "Сохранить"

✅ Уведомление об успешном обновлении

3. Отметка задачи как выполненной
Нажмите ✅ на карточке задачи

Или отметьте чекбокс в режиме редактирования

Задача переместится в "Выполненные" при фильтрации

4. Удаление задачи
Нажмите 🗑️ на карточке задачи

Подтвердите удаление

✅ Уведомление об успешном удалении

5. Фильтрация задач
Все задачи - показываются все задачи

Активные - только невыполненные задачи

Выполненные - только завершенные задачи

Особенности интерфейса
Анимации: Плавное появление/исчезновение задач

Уведомления: Информативные сообщения о действиях

Адаптивность: Работает на мобильных и десктопах

Валидация: Проверка введенных данных

🔧 Детальная настройка
Настройка бэкенда
Порт сервера
По умолчанию бэкенд запускается на порту 8000. Для изменения порта:

python
# В backend/run.py измените:
uvicorn.run("app.main:app", host="0.0.0.0", port=8000, reload=True)
# На нужный порт, например 8001:
uvicorn.run("app.main:app", host="0.0.0.0", port=8001, reload=True)
База данных
По умолчанию используется SQLite. Файл БД создается автоматически в backend/database.db.

Для сброса базы данных:

bash
cd backend
rm database.db
python run.py
Настройка фронтенда
Порт разработки
По умолчанию фронтенд запускается на порту 5173. Для изменения в frontend/vite.config.js:

javascript
export default defineConfig({
  server: {
    port: 5173, // Измените на нужный порт
    host: true
  }
})
Изменение API адреса
Если бэкенд запущен на другом порту или хосте, измените в frontend/src/services/api.js:

javascript
const apiClient = axios.create({
  baseURL: 'http://localhost:8000', // Измените на ваш URL
  // ...
});

  -d '{"title": "Тестовая задача", "description": "Описание тестовой задачи"}'

curl -X DELETE http://localhost:8000/tasks/1

# Отметить задачу как выполненную
curl -X PATCH http://localhost:8000/tasks/1/complete

🔍 Поиск и устранение неисправностей
Проблема: Бэкенд не запускается
Решение:

bash
# Проверьте, что Python установлен
python --version

# Проверьте, что виртуальное окружение активировано
# (В командной строке должно быть (venv))

# Проверьте, что зависимости установлены
pip list | grep fastapi

# Проверьте, что порт 8000 свободен
netstat -ano | findstr :8000  # Windows
lsof -i :8000                 # Linux/Mac
Проблема: Фронтенд не подключается к бэкенду
Решение:

Убедитесь, что оба сервера запущены

Проверьте CORS настройки в backend/app/main.py:

python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # Убедитесь, что есть эта строка
    # ...
)
Проверьте консоль браузера (F12) на наличие ошибок CORS

Проблема: Нет анимаций или уведомлений
Решение:

Убедитесь, что файл frontend/src/assets/styles/animations.css существует

Проверьте, что он импортирован в main.js

Проверьте консоль браузера на наличие ошибок загрузки CSS

Проблема: База данных не создается
Решение:

bash
cd backend
# Удалите старый файл БД
rm -f database.db
# Перезапустите бэкенд
python run.py
📊 API документация
Основные эндпоинты
GET /
Проверка работоспособности API

GET /tasks
Получение списка задач с пагинацией

Параметры:

skip (int, optional): Пропустить N задач (по умолчанию 0)

limit (int, optional): Ограничить количество (по умолчанию 100, максимум 1000)

completed (bool, optional): Фильтр по статусу выполнения

POST /tasks
Создание новой задачи

Тело запроса (JSON):

json
{
  "title": "Заголовок задачи",
  "description": "Описание задачи (опционально)"
}
GET /tasks/{id}
Получение задачи по ID

PUT /tasks/{id}
Обновление задачи

Тело запроса (JSON):

json
{
  "title": "Новый заголовок",
  "description": "Новое описание",
  "is_completed": true
}
DELETE /tasks/{id}
Удаление задачи

PATCH /tasks/{id}/complete
Отметить задачу как выполненную
